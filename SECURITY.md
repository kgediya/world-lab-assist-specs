# Security Policy

## Reporting a Vulnerability

If you discover a security issue, please avoid opening a public issue with sensitive details.

Instead, report it privately to the project maintainer with:
- a description of the issue
- steps to reproduce it
- potential impact
- any suggested mitigation

## Notes for This Prototype

This project currently allows local persistence of a user-provided API key for prototype convenience. For a production deployment, consider moving to a server-side saved-key or session-token architecture.
